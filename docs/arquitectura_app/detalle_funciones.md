+--------------+------------------------------------------------------------+
| Módulo       | Detalle                                                    |
+==============+============================================================+
| Recepción    | 1.  El usuario realizará el registro en el aplicativo de   |
|              |     SDRERC de los documentos recibidos mediante el SITD    |
|              |     (Rect. Adm., reconstitución, actualización de datos,   |
|              |     título de nacionalidad, apelación, cancelación,        |
|              |     reconsideración). El registro en SDRERC puede ser      |
|              |     individual o mediante la carga diaria; donde los       |
|              |     campos obligatorios serán:                             |
|              |                                                            |
|              |     - Tipo de Solicitud.                                   |
|              |                                                            |
|              |     - Fecha de Solicitud.                                  |
|              |                                                            |
|              |     - Solicitado por.                                      |
|              |                                                            |
|              |     - Documento de Identidad del Solicitante.              |
|              |                                                            |
|              |     - N.º Trámite Web.                                     |
|              |                                                            |
|              |     - Tipo de Documento.                                   |
|              |                                                            |
|              |     - Tipo de Acta.                                        |
|              |                                                            |
|              |     - N.º Acta.                                            |
|              |                                                            |
|              |     - Nombre del Titular.                                  |
|              |                                                            |
|              | 2.  El sistema realizará una validación de duplicidad      |
|              |     teniendo como criterio (número de acta y nombre del    |
|              |     titular).                                              |
|              |                                                            |
|              | 3.  El usuario validará la información a registrar en la   |
|              |     ventana de la previsualización y de ser necesario      |
|              |     podrá editar o modificar alguna información; luego     |
|              |     procederá con la confirmación del registro.            |
|              |                                                            |
|              | 4.  El sistema generara un contador de días pendientes de  |
|              |     atención por tipo de procedimiento (Rect. Adm. con un  |
|              |     máximo de 30 días; Reconsideración con un máximo de 15 |
|              |     días y para apelación con un máximo de 30 días)        |
|              |                                                            |
|              | 5.  El sistema generará un número de expediente para cada  |
|              |     solicitud, a excepto de identificados como posibles    |
|              |     duplicados, reconsideración y apelación estas son      |
|              |     derivadas al módulo Asignación para su evaluación      |
|              |     posterior. De ser necesario el asignador realizará la  |
|              |     asociación al expediente principal o generará el       |
|              |     número de expediente.                                  |
|              |                                                            |
|              | 6.  El sistema generará una observación o alerta como      |
|              |     posible grupo familiar, teniendo como criterio los dos |
|              |     apellidos del titular).                                |
|              |                                                            |
|              | 7.  Luego de realizado el registro el asignador también    |
|              |     podrá editar la información siempre se encuentre en    |
|              |     estado de RESGISTRADO.                                 |
+--------------+------------------------------------------------------------+
| ASIGNADOR    | 1.  EL usuario tendrá una bandeja donde podrá visualizar   |
|              |     los registros nuevos, donde primero deberá validar     |
|              |     aquellos registros que no se le generaron número de    |
|              |     expediente para confirmar si algunos registros serán   |
|              |     asociados o se generará un número de expediente.       |
|              |                                                            |
|              | 2.  El usuario registrará en el aplicativo de SDRERC la    |
|              |     información de las cartas de respuestas que ha         |
|              |     recibido mediante el SITD, para ello colocará el       |
|              |     numero de la hoja de envío en el panel del documento.  |
|              |                                                            |
|              | - Tener en cuenta los plazos por documento:                |
|              |                                                            |
|              |   - CARTA EDICTO 30 días -- para publicación 15 dìas       |
|              |                                                            |
|              |   - CARTA INDAGATORIO 10 días                              |
|              |                                                            |
|              |   - CARTA PRECISAR PRETENSION 30 días                      |
|              |                                                            |
|              |   - CARTA FALTA SUSTENTO 30 días                           |
|              |                                                            |
|              | 3.  El usuario deberá validar los registros que tengan la  |
|              |     alerta de posible grupo familiar, de esta manera los   |
|              |     asignará a un mismo abogado.                           |
|              |                                                            |
|              | 4.  El usuario deberá validar la carga de laboral que      |
|              |     tiene cada abogado (para ello se le habilitará una     |
|              |     nueva ventana).                                        |
|              |                                                            |
|              | 5.  El usuario asignará los expedientes a los abogados     |
|              |     mediante la asignación individual o masiva, para ello  |
|              |     se le habilitará un panel donde colocará el numero de  |
|              |     la hoja de envió con el cual se le estará realizando   |
|              |     la asignación mediante el SITD.                        |
|              |                                                            |
|              | Tener en cuenta que el asignador deberá identificar los    |
|              | plazos pendientes de atención.                             |
+--------------+------------------------------------------------------------+
| ANALISIS     | 1.  El usuario deberá identificar si el expediente         |
|              |     corresponde o no atenderlo, en caso de no corresponder |
|              |     deberá archivar el expediente en SDRERC y en el SITD   |
|              |     realizará la derivación a la UUOO que corresponda.     |
|              |                                                            |
|              | 2.  Para los casos que si es de su competencia inicia el   |
|              |     proceso de evaluación; validara si el expediente       |
|              |     cuenta con:                                            |
|              |                                                            |
|              |     a.  Cuenta con sustento                                |
|              |                                                            |
|              |     b.  Precisa pretensión                                 |
|              |                                                            |
|              | En caso de no contener esta información, enviara al        |
|              | ciudadano las cartas correspondientes (carta falta         |
|              | sustento, carta precisar pretensión)                       |
|              |                                                            |
|              | 3.  El usuario deberá validar si el acta se encuentra      |
|              |     INCORPORADA:                                           |
|              |                                                            |
|              |     a.  NO ESTÁ INCORPORADA. - deberá proyectar un oficio  |
|              |         derivando la atención a la municipalidad (OREC) y  |
|              |         una carta de notificación al ciudadano.            |
|              |                                                            |
|              |     b.  SI ESTÁ INCORPORADA, validar si se encuentra en    |
|              |         tablas maestras, en caso de no estar TM enviará a  |
|              |         culminación en línea a SDPRC. (EN CASO DE          |
|              |         ENCONTRAR MAS DE 1 ACTA deberá enviar al ciudadano |
|              |         la CARTA INDAGATORIA)                              |
|              |                                                            |
|              | > Luego de que el acta ya se encuentre en TM, se verifica  |
|              | > si el acta requiere RECONSTITUCION.                      |
|              |                                                            |
|              | - En caso de requerir reconstitución, se emitirá           |
|              |   IMPROCEDENTE, (proyectando informe y resolución).        |
|              |                                                            |
|              | - En caso de no requerir reconstitución y no cumple con    |
|              |   legitimidad, se emitirá IMPROCEDENTE, (proyectando       |
|              |   informe y resolución).                                   |
|              |                                                            |
|              | - En caso de no requerir reconstitución, cumplir con       |
|              |   legitimidad, se validará medios probatorios\... de no    |
|              |   cumplir con este último se declara IMPROCEDENTE          |
|              |                                                            |
|              | - En caso de no requerir reconstitución, cumplir con       |
|              |   legitimidad, cumple con los medios probatorios, y de no  |
|              |   requerir CARTA EDICTO se determina PROCEDENTE/PROCEDENTE |
|              |   EN PARTE/IMPROCEDENTE.                                   |
|              |                                                            |
|              | - En caso de no requerir reconstitución, cumplir con       |
|              |   legitimidad, cumple medios probatorios y requiere CARTA  |
|              |   EDICTO. Se emite la carta (carta intermedia) y se        |
|              |   congela el tiempo pendiente de atención.                 |
|              |                                                            |
|              | <!-- -->                                                   |
|              |                                                            |
|              | - Para todas las cartas intermedias, a no tener respuesta  |
|              |   cumplido el plazo se declara en abandono.                |
|              |                                                            |
|              | - En caso se tiene respuesta se continua con la            |
|              |   evaluación.                                              |
|              |                                                            |
|              | - Todos los documentos son elevados a verificación.        |
|              |                                                            |
|              | - El abogado, tendrá que completar información faltante    |
|              |   del registro, así como DNI, TIPOS DE DOCUMENTO DE        |
|              |   IDENTIDAD, UBICACIÓN DE NOTIFICACÓN, ECTC. también       |
|              |   tendrá que crear la carpeta del expediente digital y     |
|              |   colocará el enlace en el aplicativo SDRERC.              |
+--------------+------------------------------------------------------------+
| VERIFICACION | 1.  El supervisor de verificación validará la consistencia |
|              |     de los documentos, en caso de haber alguna observación |
|              |     devolverá los documentos al abogado (análisis) para su |
|              |     corrección. En caso de estar correcto se procede a     |
|              |     firmar. Tener en cuenta que las CARTAS EDICTOS son     |
|              |     firmados por el SUB-DIRECTOR.                          |
|              |                                                            |
|              | 2.  Solo las resoluciones (PROCEDENTE, PROCEDENTE EN PARTE |
|              |     E IMPROCEDENTES) son derivados a ejecución, los otros  |
|              |     documentos son enviados a notificación.                |
|              |                                                            |
|              | 3.  El supervisor será el encargado de cambiar los estados |
|              |     de los documentos inclusive aquellos que son firmados  |
|              |     por el SUB-DIRECTO.                                    |
+--------------+------------------------------------------------------------+
| EJECUCION    | 1.  El supervisor extraerá del SITD las resoluciones       |
|              |     firmadas, y validara que no tenga error material, en   |
|              |     caso de tener error revertirá el documento con una     |
|              |     hoja de envío al abogado (análisis) para su            |
|              |     subsanación, el cual nuevamente se enviara a           |
|              |     verificación y luego a firmar. Tener en cuenta que se  |
|              |     emitirá un nuevo número de resolución.                 |
|              |                                                            |
|              | Para las resoluciones que no tienen error material, los    |
|              | derivará al mismo abogado que analizo el expediente, este  |
|              | lo derivará en la bandeja de ejecución.                    |
|              |                                                            |
|              | 2.  En la bandeja de ejecución del abogado, para los casos |
|              |     IMPROCEDENTES, proyectara la carta de notificación.    |
|              |     Para los casos de PROCEDENTE O PROCEDENTE EN PARTE     |
|              |     realizara la anotación textual y proyectara la carta   |
|              |     de notificación. En los tres casos la carta de         |
|              |     notificación será validada por el supervisor.          |
+--------------+------------------------------------------------------------+
| NOTIFICACION | 1.  EL SUPERVISOR verificara la carta de notificación en   |
|              |     caso de haber error o inconsistencia se devolverá ala  |
|              |     abogado de ejecución para su subsanación, luego lo     |
|              |     validara y firmara la carta de notificación.           |
|              |                                                            |
|              | 2.  Validara la bandeja de documentos que se tiene para    |
|              |     notificar y derivara dichos documentos a los abogados  |
|              |     de notificación.                                       |
|              |                                                            |
|              | 3.  EL ABOGADO de notificación se realizará el proceso de  |
|              |     notificación detallando por documento en el panel.     |
|              |     TIPO DE NOTIFICACION, FECHA DE NOTIFICACION, ¿TIENE    |
|              |     ACUSE?                                                 |
|              |                                                            |
|              | Tener en cuenta que se puede notificar hasta 3 veces. (1   |
|              | virtual y 2 presencial o en físico).                       |
