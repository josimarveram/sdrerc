En la fecha señalada, se llevó a cabo una reunión presencial entre
representantes de la Dirección de Registros Civiles (DRC) y la
Subdirección de Recursos Registrales (SDRERC), con la finalidad de
presentar los avances del aplicativo para la gestión de expedientes y
recoger validaciones y observaciones por parte de los usuarios.

| **TEMAS TRATADOS**                                                    |
+:======================================================================+
| 1.  Se realizó la revisión de los puntos establecidos en el Acta N.º  |
|     011-2026, identificándose las siguientes mejoras y adecuaciones:  |
|                                                                       |
| <!-- -->                                                              |
|                                                                       |
| a.  Módulo de Recepción                                               |
|                                                                       |
| - Se deberá añadir un combo para seleccionar el tipo de documento del |
|   remitente y del titular. (RUC / Pasaporte / Carnet De Extranjería / |
|   DNI / NA).                                                          |
|                                                                       |
| - Importación de carga diaria                                         |
|                                                                       |
|   - El sistema deberá generar alertas para los registros duplicados,  |
|     considerando como criterios de validación el número de acta y el  |
|     nombre del titular.                                               |
|                                                                       |
|   - El sistema deberá permitir la importación de todos los registros  |
|     incluyendo aquellos identificados como duplicados. Las alertas    |
|     tendrán carácter informativo y no restrictivo.                    |
|                                                                       |
|   - El número de expediente deberá generarse automáticamente durante  |
|     la importación de la carga diaria, utilizando un esquema de       |
|     correlativo anual.                                                |
|                                                                       |
| b.  Módulo de Expedientes por Trabajar                                |
|                                                                       |
| - Se deberá añadir una opción de selección para validar si            |
|   corresponde o no seguir con el análisis del expediente. En caso de  |
|   no corresponder, se habilitará un campo para ingresar el motivo.    |
|                                                                       |
| c.  Módulo de Expedientes por Verificar                               |
|                                                                       |
| - Se requiere añadir una opción donde el supervisor pueda dar como    |
|   "archivado" las solicitudes o registros que fueron rechazados por   |
|   el abogado en la etapa de análisis (según el punto b).              |
|                                                                       |
| d.  Módulo Band. Ejecución y notificación                             |
|                                                                       |
| - Deberá permitir importar un archivo Excel con nuevos campos para la |
|   actualización masiva de información.                                |
|                                                                       |
| 2.  Otras mejoras identificadas:                                      |
|                                                                       |
| - El sistema deberá permitir realizar la selección múltiple para la   |
|   asignación masiva de abogados en los módulos de asignación, bandeja |
|   de ejecución y bandeja de notificación.                             |
|                                                                       |
| - Se deberá añadir la opción de "abandono" en la lista de estados de  |
|   notificación.                                                       |
|                                                                       |
| - Se deberá incorporar paginación en todas las listas, bandejas y     |
|   módulos del sistema, a fin de mejorar la visualización y            |
|   ordenamiento de la información.                                     |
|                                                                       |
| 3.  Información pendiente por parte de SDRERC.                        |
|                                                                       |
| - Remisión del archivo de Excel para carga diaria con información     |
|   actualizada (estado de ejecución y numero de resolución) para la    |
|   bandeja de ejecución y notificación.                                |
|                                                                       |
| 4.  Revisión de flujos pendientes:                                    |
|                                                                       |
| - Validar el fujo del proceso de ejecución y notificación.            |
|                                                                       |
| - Definir la opción "Asignación de respuesta".                        |
|                                                                       |
| - Definir el flujo del proceso de expediente digital.                 |
