En la fecha señalada, se llevó a cabo una reunión de trabajo presencial
entre los representantes de la Dirección de Registros Civiles (DRC) y la
Subdirección de Recursos Registrales (SDRERC) para presentar los avances
en el desarrollo del aplicativo informático para la gestión de
expedientes, así como recabar las validaciones, observaciones y ajustes
en los requerimientos operativos para optimizar las etapas de Ejecución
y Notificación.

| **TEMAS TRATADOS**                                                    |
+:======================================================================+
| 1.  Se revisaron las funcionalidades del aplicativo, con énfasis en   |
|     las etapas de ejecución y notificación, abarcando asimismo las    |
|     opciones de recepción, asignación, análisis, asociación de        |
|     expedientes duplicados, cargos de acuse, expediente digital y     |
|     publicación, con lo cual se da por culminada la revisión del      |
|     flujo correspondiente a la SDRERC.                                |
|                                                                       |
| 2.  La SDRERC informó que se implementarán ajustes operativos, ya que |
|     el análisis y la ejecución de cada solicitud estará a cargo de un |
|     único abogado durante todo el proceso, eliminándose la asignación |
|     por etapas independientes. Por tanto, la "Bandeja de Ejecución\"  |
|     ya no requerirá nueva asignación; el expediente permanecerá       |
|     vinculado al abogado asignado inicialmente.                       |
|                                                                       |
| 3.  Asimismo, se indicaron los siguientes ajustes:                    |
|                                                                       |
| *Gestión de expedientes*                                              |
|                                                                       |
| a.  Módulo de Recepción                                               |
|                                                                       |
| - Se ratificó que el proceso de importación de la carga diaria debe   |
|   incluir la totalidad de los registros, incluso aquellos             |
|   identificados como duplicados. La revisión y el cruce con el        |
|   documento primigenio se gestionarán en el módulo de \"Asignación\". |
|                                                                       |
| b.  Módulo de Asignación                                              |
|                                                                       |
| - El asignador será el responsable de asociar los registros           |
|   duplicados, cartas u otros documentos, validando su relación con el |
|   documento principal y, de ser el caso, asociarlo al expediente      |
|   principal. Estos casos serían derivados a la "Bandeja de            |
|   notificación", sin requerir el flujo regular del proceso.           |
|                                                                       |
| c.  Módulo de Expedientes por Trabajar                                |
|                                                                       |
| - Campo "Corresponde", "No corresponde": Al seleccionar               |
|   "Corresponde", se aplicaría el flujo completo de análisis,          |
|   ejecución y notificación. Si se marca \"No corresponde\", se        |
|   habilitará el campo "Motivo" para su registro; y el abogado podrá   |
|   archivar el expediente y, mediante el trámite documentario,         |
|   derivarlo al área correspondiente.                                  |
|                                                                       |
| - Campo \"Resultado\": Se mantendrán las opciones actuales            |
|   ("Procedente", "Improcedente" y "Procedente en parte"), las cuales  |
|   continuarán el flujo regular. Se incorporarán los valores "En       |
|   abandono" y "Observación administrativa", que serán derivadas       |
|   directamente a la \"Bandeja de Notificación\".                      |
|                                                                       |
| - Los campos: "¿tiene observación?" y "Tipo de observación", serán de |
|   modo lectura; solo se habilitarán en el módulo "Expediente por      |
|   verificar".                                                         |
|                                                                       |
| - Se incluirá el campo: "Incorporado (Sí / No)" de carácter           |
|   obligatorio que permita identificar si el acta se encuentra o no    |
|   incorporada. Si se marca "Sí" se podrá marcar las casillas:         |
|   "Reconstitución", "Legitimidad" y "Si cumple con medios             |
|   probatorios", caso contrario estarán deshabilitados.                |
|                                                                       |
| - Obligatoriedad de datos: El registro del correo electrónico será de |
|   carácter obligatorio para todos los trámites virtuales.             |
|                                                                       |
| - Documentos analizados: se habilitará esta sección para registrar    |
|   los datos: Fecha, Tipo de documento (Carta / Oficio / Memo u otro   |
|   que corresponda), descripción y su estado (en proyecto, en despacho |
|   y emitido).                                                         |
|                                                                       |
| *Ejecución y notificación*                                            |
|                                                                       |
| d.  Módulo de Bandeja de Ejecución                                    |
|                                                                       |
| - Se podrá revertir el estado de un documento cuando el supervisor    |
|   identifique observaciones, manteniendo la trazabilidad de la        |
|   corrección y evitando que el expediente continúe a ejecución o      |
|   notificación con documentos pendientes o inconsistentes.            |
|                                                                       |
| - Una vez firmado el documento, el supervisor deberá registrar el     |
|   número de la resolución en el campo establecido, de manera que      |
|   dicho dato se encuentre disponible para las etapas posteriores.     |
|                                                                       |
| e.  Módulo de ejecución por trabajar                                  |
|                                                                       |
| - Se incluirá el estado *Indagatorio* dentro de las opciones de       |
|   ejecución.                                                          |
|                                                                       |
| - Documentos analizados: se habilitará esta sección para registrar    |
|   los datos: Fecha, Tipo de documento (Carta / Oficio / Memo u otro   |
|   que corresponda), descripción y su estado (en proyecto, en despacho |
|   y emitido).                                                         |
|                                                                       |
| f.  Módulo notificación por trabajar (Bandeja de Cargos de Acuse):    |
|                                                                       |
| - Se definió que el flujo estándar consta de una (1) notificación     |
|   virtual y dos (2) presenciales. Agotadas estas instancias sin       |
|   éxito, se procederá con la generación de la publicación.            |
|                                                                       |
| - Se evaluará la implementación o adecuación de una bandeja para      |
|   registrar cargos de acuse. Para ello, la SDRERC enviará la matriz   |
|   en Excel con las variables y datos utilizados para el seguimiento   |
|   de las notificaciones.                                              |
|                                                                       |
| - Se evaluará la implementación o adecuación de un módulo de          |
|   publicaciones, con información mínima que permita relacionar los    |
|   casos sin acuse con el expediente. Para ello, la SDRERC enviará la  |
|   matriz en Excel con la estructura de datos para las publicaciones.  |
|                                                                       |
| 4.  Otras coordinaciones:                                             |
|                                                                       |
| - Sobre el expediente digital, la SDRERC va a coordinar si aplicará   |
|   la asignación de los abogados para esta etapa. Dado que el código   |
|   se genera desde las etapas iniciales, se recomienda que el abogado  |
|   responsable cree la carpeta del expediente digital y registre el    |
|   enlace en el módulo "Expediente por trabajar"; siendo además        |
|   responsable de la carga de los documentos en la ruta compartida     |
|   correspondiente.                                                    |
