En la fecha señalada, se llevó a cabo una reunión presencial entre
representantes de la Dirección de Registros Civiles (DRC) y la
Subdirección de Recursos Registrales (SDRERC), con la finalidad de
presentar los avances del aplicativo para la gestión de expedientes y
recoger validaciones y observaciones por parte de los supervisores.

| **TEMAS TRATADOS**                                                    |
+:======================================================================+
| 1.  Se identificaron las siguientes mejoras en la revisión del        |
|     aplicativo:                                                       |
|                                                                       |
| <!-- -->                                                              |
|                                                                       |
| a.  Módulo de Recepción                                               |
|                                                                       |
| - El registro debe contemplar los siguientes campos mínimos: tipo y   |
|   número de acta, tipo de solicitud, canal de ingreso y DNI y nombres |
|   del titular                                                         |
|                                                                       |
| - Los demás campos no deberán ser obligatorios en esta etapa (por     |
|   ejemplo, grupo familiar, datos de notificación como correo, celular |
|   y dirección), ya que podrán ser completados durante el análisis.    |
|                                                                       |
| - Para actas de matrimonio, se deberá permitir el registro de dos     |
|   titulares.                                                          |
|                                                                       |
| b.  Carga masiva de información                                       |
|                                                                       |
| - Se requiere habilitar una opción de carga diaria mediante archivo   |
|   Excel que generan desde el SITD. Este archivo contendrá los datos   |
|   básicos de la solicitud e incluso permitirá completar información   |
|   adicional (evitaría el registro manual uno a uno).                  |
|                                                                       |
| - Considerar el formato con el nuevo SITD (actualmente en marcha      |
|   blanca).                                                            |
|                                                                       |
| c.  Validación de duplicidad                                          |
|                                                                       |
| - El sistema deberá validar registros duplicados y generar alertas    |
|   (Se recomienda validar con el número de acta y el DNI del titular). |
|                                                                       |
| - Este control también deberá aplicarse en los procesos de carga      |
|   masiva diaria, contrastando con la información existente en el      |
|   sistema.                                                            |
|                                                                       |
| d.  Generación de número de expediente                                |
|                                                                       |
| - Se recomienda que el número de expediente se genere en la etapa de  |
|   análisis (módulo "Expediente por trabajar"), utilizando un esquema  |
|   de correlativo anual.                                               |
|                                                                       |
| e.  Módulo de Expedientes por Trabajar                                |
|                                                                       |
| - Se deberá incluir los campos: número de documento y su estado para  |
|   cada tipo de documento que generan.                                 |
|                                                                       |
| - Confirmar si el supervisor actualizaría el estado de cada documento |
|   emitido (o tal vez es el propio abogado que realizó el análisis) -- |
|   definir la responsabilidad.                                         |
|                                                                       |
| f.  Otras mejoras identificadas:                                      |
|                                                                       |
| - Exportar reportes en formato Excel para las etapas del proceso.     |
|                                                                       |
| - Incorporar un mantenimiento de "Descripción por tipo de documento", |
|   con valores preconfiguradas y breves en el módulo "Expedientes por  |
|   trabajar".                                                          |
|                                                                       |
| - Evaluar la incorporación de la opción de gestión de documentos para |
|   la firma del subdirector o sería en el módulo "Expedientes por      |
|   trabajar".                                                          |
|                                                                       |
| - Incluir un campo que muestre los días restantes para la atención de |
|   documentos. Se sugiere configurar un plazo máximo por tipo de       |
|   documento en el mantenimiento, que servirá como base para el        |
|   cálculo automático de los días pendientes.                          |
|                                                                       |
| g.  Temas de desarrollo por verificar                                 |
|                                                                       |
| - Se debe verificar el correcto funcionamiento cuando se ingresa con  |
|   algún rol ya que no se mostraba lista de documentos por trabajar.   |
|                                                                       |
| h.  Información pendiente por parte de SDRERC                         |
|                                                                       |
| - Archivo Excel para carga inicial (se recomienda con información     |
|   desde el 2022)                                                      |
|                                                                       |
| - Archivo Excel para carga diaria (la que descargan y completan       |
|   actualmente)                                                        |
|                                                                       |
| - Lista de estados utilizados en cada módulo (según formato actual)   |
|                                                                       |
| - Lista de tipos de documentos manejados en la MPV (como mínimo, se   |
|   deberá replicar esta información)                                   |
|                                                                       |
| - Definición de roles por módulo                                      |
|                                                                       |
| - Definición de la descripción por tipo de documento                  |
|                                                                       |
| - Definición de la estructura del número de expediente.               |
|                                                                       |
| - Plantillas de los documentos                                        |
|                                                                       |
| i.  Revisión de flujos pendientes:                                    |
|                                                                       |
| - Definir el fujo del proceso de notificación                         |
|                                                                       |
| - Definir la opción "Asignación de respuesta" (mencionado hoy).       |
|                                                                       |
| - Establecer los reportes mínimos requeridos (por ejemplo, para       |
|   trazabilidad)                                                       |
